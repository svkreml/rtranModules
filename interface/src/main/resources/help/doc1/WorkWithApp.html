Работа с программой

Этап 1 - Создание (загрузка) проекта

С помощю меню или кнопок расположеных на панели инструментов (слева), создаём или загружаем проект.

 

Этап 2 - Управление данными

После загрузки, в окне управления проектом появляются элементы управления данными проекта (данные, алгоритм, алфавит и память). 

Данные

При двойном нажатии по элементу "Данные", появляется диалоговое окно, предлогающее вставить входные данные и символ конца. Так же присутствует возможность загрузки входных данных из файла, с помощю кнопки "Открыть файл".

Рисунок 1 - Диалоговое окно "Редактор входной ленты"

 

Алфавит

Алфавит является некоторой интегральной характеристикой обрабатываемой информации. Примерами алфавитов могут быть цифры, буквы, буквы и цифры и тд.

При двойном нажатии по элементу "Алфавит",  в табличном представлении показывается таблица с наборами алфавитов.

Рисунок 2 - Таблица алфавитов

На панели инструментов (снизу) расположены кнопки:

 - Добавление алфавита

 - Редактирование алфавита

 - Удаление алфавита 

С помощью кнопки "Добавить" пользователь может создать свой собственный алфавит. Также, при нажатии кнопки "Редактировать" появится диалоговое окно "Редактор алфавитов", в котором можно редактировать алфавит.

Алгоритм

Работа RBM осуществляется по R-программе. R-программа состоит из именованных комплексов команд. Каждая команда состоит из 4 полей: 

  • поле имени комплекса (т.н. номер вершины), пропускается, для второго и последующих операторов в теле одной команды
  • поле условия (или предиката), пропускается, для второго и последующих операторов в теле одной команды
  • поле операторов команды
  • поле преемника ( номер конечной вершины, заполняется только для последнего оператора).

При двойном нажатии по элементу "Алгоритм",  в табличном представлении показывается таблица с набором последователностей алгоритма.

Рисунок 4 - Таблица алгоритма

На панели инструментов (снизу) расположены кнопки:

 - Добавить вершину

 - Редактировать вершину

 - Удалить вершину

С помощью кнопки "Добавить вершину", открывается диалоговое окно "Редактор Алгоритма", где пользователь может создать элемент алгоритма. Так же, при двойном щелчке по строке вершины алгоритма появится диалоговое окно "Редактор Алгоритма", в котором можно задавать условия и действия для данной вершины.

Типы памятей:

Регистровая память – тип памяти, который удобно представить в виде бесконечной в одну сторону ленты

Рисунок 8 - Регистровая память

Вагон - тип памяти, который удобно представить в виде бесконечной в обе стороны ленты на которой в любом месте сделана некоторая упорядоченная, не содержащая пропусков запись.

Рисунок 9 - Вагон

В случае вагонной памяти указывается, откуда идет считывание (запись) данных для этого к имени памяти приписывается буква L или R (слева или справа соответственно), например, если имя памяти «V», то обозначение того, что будет вестись работа с правой частью памяти, выглядит следующим образом: «RV».

Счетчиковая память 

Этот тип памяти представлен бесконечной влево лентой, на которой справа налево может быть записано любое число. Слева стаошие разряды, справа младшие. Запись в счетчиковую память осуществляется по арифметическому оператору присваивания. Чтение осуществляется как одного неделимого слова, аналогично регистру.

Табличная память - этот тип памяти представляется бесконечной в обе стороны лентой, разбитой на ячейки. Ячейки табличной памяти имеют структуру, аналогичную структуре ячеек вагонной памяти.В отличие от вагонной памяти, записи в ячейках табличной памяти не упорядочены и в табличной памяти нет заранее фиксированных доступных ячеек.

Этап 3 - Запуск проекта

Запуск проекта предназначен для интерпретации пользовательской программы, написанной на Р-коде, и включает в себя соответственно окно выдачи результатов работы алгоритма на Р-коде и механизм интерпретации программы.
